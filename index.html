CSS Syntax

A sintaxe do CSS consiste em um seletor e um bloco de declaração.

Estrutura Básica:

seletor {
    propriedade: valor;
    propriedade: valor;
}

Exemplo Prático:

Este texto está estilizado
.syntax-demo {
    color: blue;
    font-size: 18px;
    background-color: #f0f0f0;
}

How to Add CSS

Existem três maneiras de adicionar CSS ao HTML:

1. CSS Inline

Aplicado diretamente no elemento HTML

<h1 style="color: red;">Título</h1>

2. CSS Interno

Definido dentro da tag <style> no <head>

<style>
h1 {
    color: red;
}
</style>

3. CSS Externo

Arquivo .css separado linkado no HTML

<link rel="stylesheet" href="style.css">

CSS Errors

Erros comuns em CSS e como evitá-los:

❌ Erro: Ponto e vírgula ausente

/* ERRADO */
p {
    color: red
    font-size: 16px;
}
/* CORRETO */
p {
    color: red;
    font-size: 16px;
}

❌ Erro: Chaves não fechadas

/* ERRADO */
.container {
    width: 100%;
    /* chave de fechamento ausente */
/* CORRETO */
.container {
    width: 100%;
}

CSS Box Model

O Box Model CSS descreve as caixas retangulares que são geradas para elementos:

CONTENT
O conteúdo real

Amarelo: Content | Marrom: Border | Transparente: Padding e Margin 

/* Box model padrão */
.box {
    width: 300px;
    height: 200px;
    padding: 20px;
    border: 10px solid #333;
    margin: 15px;
}

/* Box-sizing para controlar o modelo */
.border-box {
    box-sizing: border-box; /* width inclui padding e border */
}

.content-box {
    box-sizing: content-box; /* width só do conteúdo (padrão) */
}

CSS Max-Width

A propriedade max-width define a largura máxima de um elemento:

Sem Max-Width

Este elemento ocupa 100% da largura disponível, independente do tamanho da tela.

Com Max-Width: 300px

Este elemento nunca será maior que 300px, mesmo em telas grandes.

/* Largura máxima */
.container {
    width: 100%;
    max-width: 800px;
    margin: 0 auto; /* Centralizar */
}

/* Largura responsiva */
.responsive-box {
    width: 100%;
    max-width: 600px;
    min-width: 300px;
}

CSS Overflow

A propriedade overflow controla o que acontece quando o conteúdo excede o tamanho do elemento:

Overflow: Visible

Este texto é muito longo e vai transbordar para fora da caixa, sendo completamente visível mesmo que exceda os limites definidos.

Overflow: Hidden

Este texto é muito longo e vai ser cortado quando exceder os limites da caixa, ficando oculto.

Overflow: Scroll

Este texto é muito longo e vai mostrar barras de rolagem para permitir ver todo o conteúdo.

Overflow: Auto

Este texto é muito longo e mostrará barras de rolagem apenas quando necessário.

/* Valores de overflow */
overflow: visible; /* Padrão - conteúdo visível */
overflow: hidden;  /* Corta o conteúdo */
overflow: scroll;  /* Sempre mostra scroll */
overflow: auto;    /* Scroll quando necessário */

/* Overflow horizontal e vertical */
overflow-x: hidden;
overflow-y: scroll;

CSS !important

Como e quando usar a declaração !important (use com cuidado!):

⚠️ Aviso Importante:

!important deve ser usado apenas como último recurso. Ele quebra o fluxo natural de especificidade e pode tornar o CSS difícil de manter.

Exemplo de Uso (e Problema):

Elemento com !important
Este texto é amarelo devido ao !important, mesmo que outras regras tentem alterá-lo.
Hierarquia com !important
Tamanho forçado com !important

✅ Casos Válidos para !important:

  • Sobrescrever estilos inline (quando não pode alterá-los)
  • Classes utilitárias (como .hidden, .text-center)
  • Sobrescrever CSS de bibliotecas externas
  • Estilos de impressão

❌ Problemas com !important:

  • Torna o CSS difícil de manter
  • Quebra a cascata natural
  • Pode criar "guerra" de !important
  • Dificulta debugging
/* ❌ USO INCORRETO do !important */
.button {
    background: blue !important;     /* Desnecessário */
    color: white !important;         /* Desnecessário */
    padding: 10px !important;        /* Desnecessário */
}

/* ✅ USO CORRETO - alternativas ao !important */

/* 1. Aumentar especificidade naturalmente */
.header .navigation .menu .button {
    background: blue;                /* Mais específico que .button */
}

/* 2. Usar seletores mais específicos */
body .container .button {
    background: blue;
}

/* 3. Reorganizar ordem no CSS */
.button { background: red; }
.special-button { background: blue; } /* Vence por estar depois */

/* ✅ USO VÁLIDO do !important */

/* Classes utilitárias */
.hidden {
    display: none !important;        /* Sempre esconde elemento */
}

.text-center {
    text-align: center !important;   /* Sempre centraliza */
}

.no-margin {
    margin: 0 !important;            /* Remove todas as margens */
}

/* Sobrescrever bibliotecas externas */
.bootstrap-override {
    color: red !important;           /* Sobrescreve Bootstrap */
}

/* Estilos de impressão */
@media print {
    .no-print {
        display: none !important;    /* Nunca imprime */
    }
    
    .print-black {
        color: black !important;     /* Sempre preto na impressão */
    }
}

/* Sobrescrever inline styles quando necessário */
.force-color {
    color: blue !important;          /* Sobrescreve style="color: red;" */
}

/* Como evitar !important: */

/* 1. Planeje a especificidade desde o início */
/* 2. Use metodologias como BEM */
/* 3. Evite IDs para estilos */
/* 4. Organize bem a ordem do CSS */
/* 5. Use CSS custom properties (variáveis) */

:root {
    --primary-color: blue;
}

.button {
    background: var(--primary-color); /* Fácil de sobrescrever */
}

.special .button {
    --primary-color: red;            /* Sobrescreve a variável */
}

CSS Performance

Dicas para otimizar a performance do CSS:

✅ Boas Práticas para Performance:

  • Minimize CSS: Remova espaços e comentários em produção
  • Use seletores eficientes: Evite seletores muito específicos
  • Prefira classes: Ao invés de IDs ou elementos
  • Evite !important: Dificulta otimizações
  • Use CSS Grid/Flexbox: Ao invés de float

❌ O que evitar:

  • Seletores complexos: body div.container ul li a span
  • CSS não usado: Remova estilos desnecessários
  • Muitas animações: Can impact performance
  • Importações excessivas: @import é lento
  • Inline styles: Para elementos repetidos

⚡ Otimizações Avançadas:

Critical CSS
Carregue apenas o CSS necessário para above-the-fold
CSS Sprites
Combine múltiplas imagens em uma só
Tree Shaking
Remova CSS não utilizado automaticamente

CSS Accessibility

Como criar CSS inclusivo e acessível para todos os usuários:

✅ Princípios de Acessibilidade:

  • Contraste: Ratio mínimo de 4.5:1 para texto normal
  • Tamanho da fonte: Mínimo 16px para leitura confortável
  • Foco visível: Sempre mostrar onde está o foco
  • Movimento reduzido: Respeitar prefer-reduced-motion
  • Cores não exclusivas: Não use apenas cor para comunicar

Exemplos de Contraste:

✅ Bom Contraste (21:1)
Texto preto em fundo branco - fácil de ler
✅ Bom Contraste (5.9:1)
Texto branco em azul - adequado para leitura
⚠️ Contraste Moderado (4.8:1)
Texto escuro em amarelo - no limite aceitável

Estados de Foco Acessíveis:

Link Acessível

Indicadores Não Exclusivos de Cor:

✓ Sucesso (ícone + cor verde)
✗ Erro (ícone + cor vermelha)
⚠ Aviso (ícone + cor amarela)
